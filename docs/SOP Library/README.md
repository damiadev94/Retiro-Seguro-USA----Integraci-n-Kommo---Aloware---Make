# SOP Library

Purpose:
This repository contains a centralized, reusable library of Standard Operating Procedures (SOPs) for integration implementations (Make, Kommo, Aloware, HTTP patterns, testing, operations, etc.). The library separates implementation procedures from project-specific runbooks so teams can reuse, reference, and maintain authoritative procedures in one place.

Scope:
The SOP Library provides standardized, versioned procedure templates and SOPs grouped by domain. It is intended for engineers, integrators, and operations staff working on enterprise integrations.

When these SOPs are used:
- During implementation across integration projects to avoid duplicated instructions.
- When writing or updating runbooks that reference common procedures.
- For on-boarding and operational execution where standard procedures are required.

Related folders:
- 00 Standards: Naming, template and lifecycle standards for SOPs.
- 01 Infrastructure: Connection, webhook and environment management SOPs.
- 02 HTTP: HTTP integration patterns and handling.
- 03 Make: Make.com scenario and runtime practices.
- 04 Kommo: Kommo-specific integration SOPs.
- 05 Aloware: Aloware-specific integration SOPs.
- 06 Testing: Testing and validation procedures.
- 07 Operations: Deployment, monitoring and incident SOPs.
- 99 Archive: Deprecated or superseded SOPs.
