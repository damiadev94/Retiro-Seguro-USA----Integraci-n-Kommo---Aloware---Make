# 1. Purpose

## Objective

Synchronize communication activities generated in Aloware into Kommo to maintain a complete customer interaction history.

This scenario ensures that calls, SMS messages, voicemails, recordings and AI-generated summaries are automatically registered in Kommo as part of the customer timeline, providing sales representatives with complete communication visibility without requiring manual updates.

---

# 2. Business Capability

**Communication Synchronization**

---

# 3. Description

S02 implements the business process responsible for synchronizing communication events from Aloware into Kommo.

The scenario receives communication events generated by Aloware, validates the event type, identifies the corresponding business entity in Kommo, transforms the communication data into the Kommo activity model and creates the appropriate Note associated with the Lead or Contact.

Depending on the event type, the synchronization may include call metadata, SMS content, recording links and AI-generated summaries.

---

# 22. Business Rules

- Every supported communication event shall generate a single corresponding Note.
- Communication events shall only be associated with existing synchronized entities.
- Communication history shall never modify CRM business data.
- Recording links shall only be added when available.
- AI Summaries shall only be added when generated.
- Unsupported communication events shall be ignored.
- Duplicate events shall not generate duplicate Notes.
- All communication timestamps shall preserve the original event time.
